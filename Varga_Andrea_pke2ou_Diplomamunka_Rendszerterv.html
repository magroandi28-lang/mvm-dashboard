I. Modul — Adatgyűjtés és adatfeldolgozás

Technológia

PythonpandasopenpyxlGoogle ColabKSH STADAT APIECB APIENTSO-E API

Adatminőségi kihívások és megoldások

  • Heterogén xlsx struktúra: MEKH háromszor változtatta a fájlformátumát 2014–2024 között → négy különböző beolvasó függvény
  • Duplikált adatok: minden éves MEKH fájlban 2016-tól szerepelnek adatok → dátum szerinti szűrés
  • Hiányzó 2024 december: a 2025-ös fájlban volt megtalálható → kereszt-fájl szűrés
  • Windows 259 karakter limit: fájlok áthelyezésével megoldva
Külső ellenőrzés: 2022-es 45,76 TWh és 2023-as 43,70 TWh egyezik a MAVIR publikált adataival.

II. Modul — Anomáliadetekció + Rezsi-gap trendelemzés

Technológia

IQRIsolation ForestRandom ForestSMOTEseabornmatplotlib

Anomáliadetekció – 3 módszer

  • IQR → 3 anomália: 2020 október, 2021 július, 2021 október
  • Isolation Forest → 14 anomália
  • Random Forest + SMOTE → 89% pontosság, 80% recall
  • Hárommódszer egyezés: 2021 július biztosan anomália (3 979 800 MWh, 24,3°C)
Rezsi-gap összegek: 2022: 402 Mrd Ft | 2023: 918 Mrd Ft (csúcsév) | 2024: 940 Mrd Ft
Energia-ár olló csúcs: 2022-ben 107 Ft/kWh

III. Modul — Predikció | PyTorch Ensemble modell

Technológia

PyTorchnn.LinearMSE lossAdam optimizerMinMaxScaler

Modellek összehasonlítása

Az összehasonlítás három klasszikus módszert mér az Ensemble PyTorch modellel szemben — ugyanazon 80/20 train/test felosztáson, időrendben. A naiv modellek a legegyszerűbb becsléseket alkalmazzák, a SARIMA klasszikus statisztikai idősor-módszer. Az Ensemble mindháromat jelentősen felülmúlja.

ModellMAE (MWh)Pontosság*Jan. MAEMegjegyzés
Naiv (holnap=ma)25095,28%—Baseline
Naiv (heti szezon)35193,39%—Baseline
SARIMA(1,1,1)(1,1,1,7)103780,43%—Csak időbeli minta
V149,5599,07%124,63Extrém hidegnél gyenge
V273,3198,62%68,28Más hónapokban gyengébb
Ensemble ✓31,0999,41%30,568× jobb a legjobb naivnál

* Pontosság számítása: 100 − (MAE / átlagos napi fogyasztás × 100). Regressziós teljesítménymutató, nem klasszikus osztályozási accuracy.

V2 — Szintetikus adataugmentáció

  • -5 és -7°C között: átlag -6,39°C / 6 458 MWh (30 szintetikus sor)
  • -7°C alatt: átlag -8,70°C / 6 513 MWh (30 szintetikus sor)
  • Valós átlagok alapján generált — nem kitalált értékek

Adat- és feature megjegyzések

  • Hőmérséklet forrása tanításkor: MEKH historikus (utólagos mért) adat — éles futásnál Open-Meteo forecast API (7 napra előre elérhető)
  • DAM árak: ENTSO-E API — ténylegesen előre ismertek a következő napra
  • MinMaxScaler UserWarning: predikciónál numpy tömb helyett pandas DataFrame átadásával kiküszöbölhető — a pontosságot nem befolyásolta
  • Negatív DAM árak: valós piaci jelenség (megújuló túltermelés) — a rendszer helyesen kezeli

CrewAI ágensek

  • Időjárás Adatgyűjtő: Open-Meteo API — Budapest, következő 7 nap hőmérséklet
  • Energiatőzsdei Adatgyűjtő: ENTSO-E API — magyarországi DAM ár, 30 napos átlag

Dashboard kártyák

  • 7 napos fogyasztás előrejelzés (MWh) — Ensemble modell
  • Becsült napi energiaköltség (Millió Ft) — ECB EUR/HUF árfolyammal
  • Hőmérséklet előrejelzés (°C)
  • Összesített kártyák: heti fogyasztás, csúcs, minimum, DAM ár, riasztás
Jövőbeli fejlesztés: adatbázis (pl. PostgreSQL) bevezetése megoldaná a historikus előrejelzések tárolását — Render platformon Google Drive hozzáférés nélkül ez nem megvalósítható.

🔗 Rendszer összekötés — Az integrációs pont

Amikor a menedzser e-mailben dashboard-ot vagy előrejelzést kér → Flowise DASHBOARD kategóriát rendel hozzá → az n8n Switch node automatikusan visszaküldi az MVM-DataInsight Render dashboard linkjét — emberi beavatkozás nélkül.

Konfabuláció mindkét szinten kizárva: Flowise csak az email szövege alapján kategorizál — a dashboard csak valós API adatokat jelenít meg.
